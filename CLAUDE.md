# tech-humanizer-skill

Turn AI-shaped prose into credible human writing without flattening useful technical language.

## Project

- **Repo**: ClydeShen/tech-humanizer-skill
- **Issue tracker**: GitHub Issues (`gh issue` CLI)
- **Validate**: `python scripts/validate.py`

## Agent skills

| Skill | When to use |
|---|---|
| `tech-humanizer-skill` | Humanize text, detect AI markers, or polish prose |
| `brainstorming` | Before implementing new features or changing behavior |
| `writing-plans` | Before any multi-step implementation task |
| `systematic-debugging` | When encountering bugs or unexpected behavior |
| `verification-before-completion` | Before claiming a task is complete |
| `gsd-code-review` | Review code changes for correctness and quality |
| `context-handover` | When context window approaches 80% |
| `session-start` | At the start of every session |
| `to-issues` | Convert discussion into GitHub Issues |
| `triage` | Triage incoming issues |
